# FuelWell Reconciliation Matrix

*The decision log. April 2026 edition.*

When the five research reports disagree, this document is where the disagreement gets resolved. Each row represents a decision area, summarizes what each report recommended, identifies points of friction, and commits to a single recommendation with confidence calibration.

The format is deliberately terse. A row is a contract, not a discussion.

## How to use this document

When you face a decision the matrix has already covered, the matrix wins. Adding a new dependency, framework, or pattern that contradicts a row requires:

1. An entry in `docs/decisions.md` explaining the deviation, with a date
2. An updated row here if the deviation is permanent
3. A re-read of `contested-choices.md` to confirm the new choice doesn't re-open a contested-choices problem

If you find yourself wanting to deviate, that's a useful signal. Write down what you're trying to do and *why* the matrix's recommendation isn't serving it. The answer is usually one of:

- The matrix is right and the impulse is wrong — proceed with the matrix
- The matrix is right but the constraint has changed — update the matrix
- The matrix is wrong — update the matrix and write the lesson into `contested-choices.md`

Confidence calibration matters. **High** means "five reports agree and the alternative would have to be substantially better to justify deviating." **Medium** means "the recommendation depends on context that may change." **Low** would mean "the matrix is leaning but ready to be overruled" — there are currently no Low rows.
